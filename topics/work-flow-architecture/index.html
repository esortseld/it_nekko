問12業務分析の図法

業務分析で見る対象を区別する

業務分析で使うWFA、CRUD図、E-R図、DMMを、何を見るためのものかで区別します。業務の流れ、データ操作、データ同士の関係、要素整理の違いを整理します。

1業務分析では、見る対象によって整理方法が変わる

業務を見直すときは、まず何を明らかにしたいのかを考えます。業務分析、業務改善、システム化の文脈では、同じ業務を扱っていても、見る対象によって使う図法や整理方法が変わります。

  • 業務がどのような順序で進むのか。
  • 業務で扱うデータにどのような操作を行うのか。
  • 業務で扱うデータ同士がどのように関係しているのか。
  • 検討テーマに関係する要素をどう広げて整理するのか。

2業務の流れを見る場合はWFA

WFAは、Work Flow Architectureの略で、業務プロセスの流れを可視化するための記述様式です。ここで見る対象は「仕事全体」という大きなまとまりではなく、特定の業務プロセスの流れです。

  • どの作業から始まるのか。
  • 次にどの作業へ進むのか。
  • どこで判断や分岐があるのか。
  • どの担当者や部門に引き継がれるのか。
  • どこで完了するのか。

3データへの操作を見る場合はCRUD図

CRUD図は、業務や処理がデータに対してどの操作を行うのかを見るための図です。CRUDは、データに対する基本操作を表します。

  • Create：作成
  • Read：参照
  • Update：更新
  • Delete：削除

CRUD図は、業務の流れそのものではなく、業務や処理とデータ操作の対応関係を見るものです。どの業務が、どのデータを作成、参照、更新、削除するのかを整理します。

4データ同士の関係を見る場合はE-R図

E-R図は、Entity-Relationship Diagramの略で、実体と関連を表す図です。顧客、注文、商品など、システムで扱うデータ同士がどのように関係しているのかを見るときに使います。

E-R図は、業務の作業順序や分岐を表すものではありません。データのまとまり、構造、関連を明らかにするための図法として区別します。

5テーマから要素を広げる場合はDMM

DMMは、Diamond Mandala Matrixの略です。中心となるテーマを置き、その周辺に関連する要素を展開していく考え方です。

DMMは、業務プロセスの流れ、データへの操作、データ同士の関係を直接表すものではありません。テーマから要素を広げ、検討すべき観点を整理するためのものです。

6学習のまとめ

業務分析で使う図法や整理方法は、図法名を暗記するだけではなく、「何を見るためのものか」で判断します。

  • 業務がどの順序で進むのかを見る → WFA
  • 業務や処理がデータに対して何をするのかを見る → CRUD図
  • データ同士がどのように関係しているのかを見る → E-R図
  • テーマから関連する要素を広げて整理する → DMM
