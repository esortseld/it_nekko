問20RFP

発注前にベンダへ何を伝えるべきか

システム調達で、目的、要件、条件をベンダに示し、提案を依頼する文書の役割を整理します。

トップ / 学習要点 / 発注前にベンダへ何を伝えるべきか
学習要点一覧へ戻る

1場面をつかむ

企業が新しいシステムを導入するとき、どのベンダに依頼するかを決めるために、複数の候補企業から提案を受けることがあります。

その際、発注側は、必要な機能、前提条件、予算、納期、運用条件などを整理し、ベンダが提案を作れるように提示します。

2結論を先に押さえる

RFPとは、Request For Proposal の略で、提案依頼書のことです。

試験では、「発注側が、システム化の要求や条件を示し、ベンダに提案を求める文書」と押さえます。

3RFPの構造を確認する

RFPは、ベンダに自由に提案してもらうための文書ではありますが、発注側の要求や制約を明確に示すことが重要です。

  • 発注側：システム化の目的、要求、条件を整理して提示する。
  • ベンダ：RFPに基づいて、実現方法、費用、スケジュールなどを提案する。
  • 比較：複数の提案を同じ前提で比較しやすくする。
  • 調達：提案内容をもとに、依頼先や契約内容の検討につなげる。

4理解の基軸

RFPかどうかは、次の三つの観点で判断します。

  • 発注側が作成する文書である。
  • システム化の要求や条件を示している。
  • ベンダに提案を求める目的で使う。

5間違えやすい文書を分ける

  • RFP：発注側が要求や条件を示し、ベンダに提案を求める文書。
  • 提案書：ベンダ側が、実現方法や費用などを発注側へ提示する文書。
  • 設計書：システムの構造、処理、画面、データなどを具体化した文書。
  • 契約書：発注側と受注側の合意内容を定める文書。

6まとめで定着させる

  • RFPは、Request For Proposal の略で、提案依頼書である。
  • 発注側が、要求や条件をベンダに示す。
  • ベンダは、RFPに基づいて提案書を作成する。
  • 試験では、「誰が作るか」と「何を求めるか」を見分ける。

補足：RFPは比較の土台になる

RFPがないまま各ベンダに提案を求めると、前提条件がそろわず、費用や機能を比較しにくくなります。

RFPで要求や条件をそろえて示すことで、各ベンダの提案を同じ基準で比較しやすくなります。
