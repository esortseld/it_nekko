マネジメント系 / 品質特性

使用性では利用者が操作するインタフェースを見る

利用者が直接触れる画面やパネルが、分かりやすく操作しやすいかを判断する考え方を整理します。

学習要点一覧へ戻る

この要点で学ぶこと

使用性は、利用者が目的を達成しやすいか、操作しやすいかに関わる品質特性です。利用者が直接操作する画面、ボタン、パネルなどのインタフェースを見るのが基本です。

試験で問われる判断軸

  • 利用者が直接操作する部分に注目する。
  • 画面表示、ボタン配置、案内、入力しやすさを見る。
  • システム内部や担当者間の接点ではなく、利用者との接点を考える。

間違えやすい選択肢・近い用語との違い

  • 使用性：使いやすさ、分かりやすさ。
  • 効率性：処理時間や資源効率。
  • 可用性：必要なとき利用できること。
  • 機能性：必要な機能を備えていること。

この問題に出会ったときの考え方

誰が操作しているかを確認します。利用者が直接触れるUIが対象なら、使用性の観点で判断します。
