システム要件定義では利用者が妥当性を確認する

開発者が要件を整理する場合でも、利用者が業務上の妥当性を確認する必要があることを整理します。

学習要点一覧へ戻る

この要点で学ぶこと

システム要件定義では、業務で本当に必要な機能や制約が反映されているかを確認します。開発者が作業を進める場合でも、利用者はレビューに参加し、業務の観点から妥当性を確認します。

現実の場面と結び付けて考える概念

  • 利用者は開発者にすべて任せきりにしない。
  • 業務上必要な要件が入っているか確認する。
  • レビューに参加し、利用者の立場で妥当性を判断する。
  • 技術的実現性や保守性だけを見る役割ではない。

混同しやすいもの

  • 利用者の役割：業務に合っているか、使う立場で確認する。
  • 開発者の役割：技術的に実現できるか、保守できるかを検討する。
  • 運用担当の役割：運用手順や運用負荷を確認する。
  • 要件定義は、完成後ではなく開発前に認識を合わせる活動である。

何を見ればこの概念だと分かるか

「利用者は何をするべきか」と聞かれたら、業務の正しさを確認する立場だと考えます。開発者が作成した要件をレビューし、利用者として妥当かを確認する選択肢を選びます。
