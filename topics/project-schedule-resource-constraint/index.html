マネジメント系 / プロジェクトマネジメント

前提作業と要員制約から最短日数を求める

作業の前後関係だけでなく、同時に使える要員数の上限を考えてスケジュールを組む考え方を整理します。

学習要点一覧へ戻る

この要点で学ぶこと

プロジェクトの最短日数は、作業の前後関係だけで決まるとは限りません。並行できる作業でも、必要な要員数の合計が上限を超える場合は、順番をずらす必要があります。

現実の場面と結び付けて考える概念

  • 各作業の前提作業を確認する。
  • 同時に実施できる候補を洗い出す。
  • 同時実施時に必要な要員数の合計を確認する。
  • 要員上限を超える場合は、どちらかを後ろにずらす。
  • 最後の作業が終わる日が最短日数になる。

混同しやすいもの

  • 前後関係だけを見ると、実際には配置できない並行作業を入れてしまう。
  • 要員数は作業期間とは別の制約である。
  • クリティカルパスの考え方に加えて、資源制約を確認する。
  • WBSは作業分解の道具であり、日数計算そのものではない。

何を見ればこの概念だと分かるか

日付の枠を作り、実施可能な作業を前から置いていきます。そのたびに、同じ期間に必要な要員数が上限を超えていないか確認します。
