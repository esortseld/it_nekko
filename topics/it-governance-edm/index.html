マネジメント系 / システム監査・ITガバナンス

新しいシステムを入れるとき、経営陣は何をするのか

新しいシステムを導入する場面で、経営陣が行うべき評価・指示・モニターを見分けます。

学習要点一覧へ戻る

場面：新しいシステムを導入する

例えば、古い勤怠システムを新しいクラウド型システムへ入れ替える場面を考えます。

実際にシステムの候補を調べたり、設定したり、運用手順を作ったりするのは、情報システム部門、人事部門、外部ベンダーなどです。しかし、会社として導入する価値があるのか、誰に責任を持たせるのか、導入後に効果をどう確認するのかは、単なる作業担当者だけで決めることではありません。

まず押さえる結論

ITガバナンスでは、経営陣がシステム作業を自分で行うわけではありません。

経営陣が行うべきことは、会社の目的に照らしてITの使い方を評価し、実行の方向を示し、導入後の結果を確認することです。外部専門家や情報システム部門は支援や実務を担いますが、経営判断そのものや最終責任まで引き受ける主体ではありません。

ITガバナンスは、IT投資を会社として管理する仕組み

ITガバナンスとは、会社がITを経営目的に合うように使うための管理の仕組みです。

ここでいうITは、パソコンやシステムそのものだけではありません。システム投資、業務改善、情報セキュリティ、外部委託、運用体制、費用対効果なども含みます。

つまり、ITガバナンスは「情報システム部門の作業管理」ではなく、「会社としてITにどうお金を使い、どう責任を持ち、どう効果を確認するか」という経営上の管理活動です。

判断に使う三つの活動

経営陣の活動は、評価・指示・モニターの三つに整理できます。試験では、選択肢に書かれた行動がどの活動に当たるかを見分けます。

評価：実行前に、現状・複数案・費用対効果・リスクなどを見て、会社にとって妥当かを判断する
指示：方針を示し、責任者・権限・予算・人員などを割り当てる
モニター：実行後に、期待した効果や目標が達成されているか確認する

評価・指示・モニターを見分ける型

  • 複数の案、現状、リスク、効果を材料に、進める方向を判断している → 評価
  • 方針を示し、責任、権限、予算、人員、実行体制を決めている → 指示
  • 実施された取組について、成果、進捗、目標達成、問題発生を確認している → モニター
  • 本来経営側が持つべき判断、責任、権限を、支援者に丸ごと渡している → 不適切

公開問題ではどう問われるか

令和8年度 ITパスポート試験 公開問題 問35では、経営陣が行う活動と、外部に任せてもよい支援業務を区別する形で問われています。

選択肢を見るときは、「誰が実務をするか」だけでなく、「会社としての判断や責任まで渡していないか」を確認します。

  • 外部専門家に比較分析を依頼すること自体は不適切ではない。
  • 情報システム戦略で想定した効果の達成状況を確認する記述は、モニターと判断する。
  • 戦略を実現するために責任者や要員を割り当てる記述は、指示と判断する。
  • 外部コンサルタントに責任と権限を全面的に委譲する記述は、不適切と判断する。

選択肢で間違えやすい点

  • 外部専門家を使うこと自体を、直ちに誤りと考えない。
  • 誤りになるのは、経営陣が持つべき責任と権限まで全面的に外部へ渡してしまう場合である。
  • 「情報を収集する」は、目的が効果確認であればモニターとして判断する。
  • 「システムの作業をする人」と「会社として責任を持つ人」を分けて考える。
  • 「比較」「分析」「検討」は、導入前に妥当性を判断するためなら評価として見る。

まとめ

  • ITガバナンスは、会社がITを経営目的に合うように使うための仕組みである。
  • 経営陣は、システム作業そのものではなく、評価・指示・モニターに責任を持つ。
  • 評価は実行前の妥当性判断、指示は実行体制の決定、モニターは実行後の確認である。
  • 情報システム部門や外部専門家は支援や実務を担うが、経営陣の責任と権限を全面的に引き受ける主体ではない。
