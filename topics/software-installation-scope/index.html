マネジメント系 / プロジェクトマネジメント・契約管理

契約に含まれないPCへ導入を求められたらどうするか

ソフトウェア導入作業で追加依頼が出たとき、契約範囲・合意・費用・責任をもとに適切な行動を判断します。

学習要点一覧へ戻る

場面：契約対象外のPCにも入れてほしいと言われた

例えば、会社の各部署にあるPCへ、新しい業務用ソフトウェアを導入する作業を考えます。

契約では、導入対象のPCがあらかじめ決められています。ところが作業中に、契約対象に含まれていないPCが見つかり、現場から「このPCにも入れてほしい」と頼まれることがあります。

まず押さえる結論

契約に含まれていないPCへ、現場判断だけで勝手に導入してはいけません。

追加で導入する場合は、発注者と受注者の間で、作業範囲、費用、納期、責任を確認する必要があります。作業の続行が困難になった場合も、発注者と合意したうえで中止や復旧を判断します。

契約範囲は、作業・費用・責任の境界線

契約範囲とは、そのプロジェクトで受注者が行うことになっている作業の範囲です。

ソフトウェア導入作業では、対象となるPC、導入するソフトウェア、作業場所、作業時間、費用負担、障害時の対応などが関係します。

契約範囲を超える作業を行うと、追加費用の扱い、作業責任、障害発生時の責任、納期への影響が不明確になります。そのため、契約外の作業は、勝手に実施するのではなく、変更として扱う必要があります。

判断に使う三つの確認ポイント

試験では、作業そのものができるかどうかよりも、契約範囲・合意・責任を守っているかが問われます。

  • 契約範囲：そのPCや作業は契約に含まれているか。
  • 合意：契約外の追加作業について、発注者と受注者で合意しているか。
  • 責任：費用、納期、障害時の責任があいまいになっていないか。

適切な行動を選ぶ型

契約に含まれるPCか確認する
→ 契約外なら、勝手に導入しない
→ 追加で導入するなら、作業範囲・費用・責任を合意する
→ 作業続行が困難なら、発注者と合意して中止・復旧を判断する

公開問題ではどう問われるか

令和8年度 ITパスポート試験 公開問題 問36では、ソフトウェア導入作業中に発生した行動のうち、適切なものを選ぶ形で問われています。

選択肢を見るときは、「頼まれた作業をしているか」ではなく、契約に含まれる作業か、契約外の追加依頼を勝手に引き受けていないか、作業中止や復旧について発注者と合意しているかを確認します。

  • 契約外のPCが見つかったため、そのPCを導入対象外とする → 契約範囲を守る行動なので適切。
  • 契約外のPCへの導入を依頼され、無償で対応する → 契約にない作業を、費用や責任を明確にせず追加しているため不適切。
  • 導入作業の続行が困難になり、発注者と合意して導入前の状態に復旧する → 合意に基づく中止・復旧なので適切。

選択肢で間違えやすい点

  • 「依頼されたから対応する」は、常に適切とは限らない。
  • 無償対応は親切に見えるが、契約管理上は不適切になることがある。
  • 契約外の作業をすると、追加費用だけでなく、障害発生時の責任範囲もあいまいになる。
  • 作業を止めること自体が不適切なのではなく、発注者と合意しているかが重要である。

まとめ

  • 導入作業では、契約に含まれる作業範囲を確認する。
  • 契約対象外のPCは、勝手に導入対象へ加えない。
  • 追加作業を行う場合は、作業範囲・費用・責任について合意が必要である。
  • 作業続行が困難な場合は、発注者と合意して中止や復旧を判断する。
