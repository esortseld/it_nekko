マネジメント系 / プロジェクトマネジメント・契約管理

作業中に追加を頼まれたとき、誰と何を確認するか

現場から頼まれた作業と、契約で引き受けた作業は同じとは限りません。登場人物を整理しながら、追加依頼を受けたときの判断を見分けます。

学習要点一覧へ戻る

まずは、状況を見てみよう

ある会社が、外部の事業者に依頼して、社内のPCへ新しい業務用ソフトウェアを導入している。

導入するPCは、契約を結ぶときに決められている。受注者の作業担当者は、その一覧に従って作業を進めている。

ところが作業中、一覧に入っていないPCを使っている現場の担当者から、「このPCにも入れてください」と頼まれた。

ソフトウェアを入れる技術はある。目の前の担当者も希望している。短時間で終わる作業かもしれない。

しかし、ここで分けて考える必要がある。

作業を頼むこと。
その作業を契約へ追加すること。
追加によって生じる費用・納期・責任を決めること。

これらは、同じ行為ではない。

このカードでは、ソフトウェア導入中の追加依頼をめぐる登場人物を整理しながら、「現場から頼まれた作業」と「契約で引き受けた作業」を見分けていく。

登場人物を整理する

この場面には、いくつかの立場がある。まずは、誰が何を扱うのかを分ける。

  • 発注者：必要な作業を依頼し、受注者と作業範囲・費用・納期などを合意する。
  • 受注者：契約で引き受けた範囲の作業を行い、範囲を超える依頼があれば発注者へ確認する。
  • 現場の利用者・担当者：業務上の要望を伝え、対象PCを案内し、導入後に利用する。
  • 作業担当者：決められた手順と対象範囲に従って、実際の導入・確認・復旧などを行う。

ここで大事なのは、現場で依頼した人が、契約を変更する権限を持っているとは限らないことである。目の前の依頼者の希望だけで、受注者が作業範囲を広げることはできない。

手掛かりになる言葉

問題文では、「契約変更」という言葉を使わずに、契約範囲を超える行動かどうかを判断させることがある。そのときは、次の言葉を手掛かりにする。

手掛かりになる言葉 確認すること
契約対象、対象一覧、作業範囲、仕様 そのPCや作業が、最初から引き受けた範囲に含まれるか
追加依頼、現場からの要望、ついでに対応 契約外の作業を、その場の判断で加えようとしていないか
無償、短時間、簡単な作業 費用の有無ではなく、範囲と責任が合意されているか
中止、復旧、導入前の状態へ戻す 受注者だけで決めず、発注者と対応を合意しているか
障害、遅延、追加費用 変更による影響と責任の所在が明確になっているか

ここで問われる契約範囲

契約範囲とは、受注者が何を行い、発注者が何に対して対価を支払うのかを定めた境界である。

ソフトウェア導入では、対象となるPCだけでなく、導入するソフトウェア、作業場所、作業時間、確認方法、障害時の対応なども範囲に関係する。

契約外のPCへソフトウェアを入れると、作業が一台増えるだけでは済まないことがある。作業時間が延びる、別の設定が必要になる、障害が起きる、納期へ影響する、といった可能性が生じる。

そのため、追加作業を行うかどうかは、現場の親切や技術的な可否だけでは決めない。発注者と受注者が、変更後の作業範囲・費用・納期・責任を確認し、合意してから扱う。

追加依頼が出たときの流れ

追加を頼まれたときは、次の順序で考える。

依頼されたPCや作業が、契約範囲に含まれるか確認する
→ 含まれなければ、その場で勝手に実施しない
→ 発注者へ追加依頼の内容と影響を伝える
→ 作業範囲・費用・納期・責任を確認する
→ 発注者と受注者が合意した内容に従って作業する

作業を続けることが難しくなった場合も同じである。受注者だけで中止や復旧方法を決めるのではなく、状況と影響を伝え、発注者と合意したうえで対応する。

現場の依頼と、契約上の決定を分ける

追加作業を頼まれたとき、目の前では「作業できるかどうか」が問題に見える。しかし、契約管理で確認するのは、それだけではない。

現場で見えていること 契約上、確認すること
利用者が作業を希望している その人に契約範囲を変更する権限があるか
技術的には作業できる 契約で引き受けた作業に含まれているか
短時間で終わりそうである 納期やほかの作業へ影響しないか
無償で対応できる 障害が起きた場合の責任は誰が負うか
元の状態へ戻す必要がある 復旧方法を誰が決定するのか

現場の担当者は、業務上の要望を伝えることができる。一方、作業範囲・費用・納期・責任を変更することは、発注者と受注者の間で決める事項である。

したがって、現場から追加を頼まれたときは、その依頼を無視するのでも、その場で引き受けるのでもない。まず契約範囲に含まれるかを確認し、範囲外であれば、追加依頼として発注者へ伝える。そのうえで、発注者と受注者が条件を確認し、合意した内容に従って作業する。

判断の中心になるのは、作業の難しさや費用の有無ではない。誰が要望を出したのか、誰が契約上の変更を決めるのかを分けることである。

このカードで見抜くこと

見るべきなのは、「頼まれた作業を実行できるか」だけではない。

現場の利用者が追加を希望していても、その依頼が契約範囲に含まれるとは限らない。契約外であれば、受注者は勝手に作業を加えず、発注者と変更内容を確認する。

ソフトウェア導入中の追加依頼では、依頼した人と契約を変更できる人を分ける。契約範囲を超えるときは、作業範囲・費用・納期・責任について、発注者と受注者が合意してから進める。
