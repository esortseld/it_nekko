契約に含まれないPCへ導入を求められたらどうするか

ソフトウェア導入作業では、作業対象・費用・責任の範囲を契約に基づいて判断します。現場で追加依頼や作業続行困難が起きたときに、何を勝手にしてはいけないのかを整理します。

ソフトウェア導入作業で起きること

システム開発プロジェクトでは、作ったシステムやソフトウェアを、利用者のPCへ導入する作業が契約に含まれることがあります。

このとき大事なのは、「どのPCに導入するのか」「どこまでが契約に含まれる作業なのか」「追加で依頼された作業を誰が承認するのか」を分けて考えることです。

現場で頼まれたからといって、契約にないPCへ勝手に導入したり、無償で追加作業を引き受けたりすると、費用・責任・品質管理の範囲があいまいになります。

例：契約対象外のPCが見つかった場合

会社の各部署にあるPCへ、新しい業務用ソフトウェアを導入する作業を考えます。

契約では、導入対象のPCがあらかじめ決められています。作業中に、契約対象に含まれていないPCが見つかった場合、そのPCへ導入してよいとは限りません。

契約対象外のPCを導入対象から外す判断は、契約範囲を守る行動です。反対に、依頼されたからといって契約対象外のPCへ無償で導入することは、契約にない作業を勝手に追加する行動になります。

このページの要点

導入作業では、作業対象・費用・責任の範囲を契約に基づいて判断します。

契約に含まれていない作業を追加する場合は、発注者と受注者の間で、作業範囲、費用、納期、責任を確認する必要があります。

作業の続行が困難になった場合は、発注者と合意したうえで中止や復旧を行うことがあります。重要なのは、現場判断だけで契約範囲を広げないことです。

契約範囲とは何か

契約範囲とは、そのプロジェクトで受注者が行うことになっている作業の範囲です。

ソフトウェア導入作業では、対象となるPC、導入するソフトウェア、作業場所、作業時間、費用負担、障害時の対応などが関係します。

契約範囲を超える作業を行うと、追加費用の扱い、作業責任、障害発生時の責任、納期への影響が不明確になります。そのため、契約外の作業は、勝手に実施するのではなく、変更として扱う必要があります。

誰が関係しているか

関係者 主な役割
発注者 導入対象や追加依頼について、受注者と合意する
受注者 契約に基づいて導入作業を行い、契約外の作業は勝手に追加しない
導入作業担当者 現場で対象PCを確認し、契約外の事象を報告する
利用部門 利用したいPCや業務上の要望を伝える。ただし契約変更を単独で決定する立場とは限らない

判断の型

導入作業の行動は、次の順番で判断します。

契約に含まれるPCか確認する
→ 契約外なら、勝手に導入しない
→ 追加で導入するなら、作業範囲・費用・責任を合意する
→ 作業続行が困難なら、発注者と合意して中止・復旧を判断する

行動ごとの考え方

  • 契約外のPCが見つかったため、そのPCを導入対象外とする → 契約範囲を守る行動なので適切。
  • 契約外のPCへの導入を依頼され、無償で対応する → 契約にない作業を、費用や責任を明確にせず追加しているため不適切。
  • 導入作業の続行が困難になり、発注者と合意して導入前の状態に復旧する → 合意に基づく中止・復旧なので適切。

試験で問われる判断軸

試験では、作業そのものができるかどうかよりも、契約範囲・合意・費用負担・責任範囲を守っているかが問われます。

  • 契約対象外のものを対象外とする → 適切
  • 契約外の作業を、合意や費用整理なしに無償で行う → 不適切
  • 作業続行が困難なとき、発注者と合意して復旧する → 適切
  • 現場の依頼だけで契約範囲を広げる → 不適切

間違えやすい点

  • 「依頼されたから対応する」は、常に適切とは限らない。
  • 無償対応は親切に見えるが、契約管理上は不適切になることがある。
  • 契約外の作業をすると、追加費用だけでなく、障害発生時の責任範囲もあいまいになる。
  • 作業を止めること自体が不適切なのではなく、発注者と合意しているかが重要である。

公開問題での問われ方

この知識は、ソフトウェア導入作業中に発生した行動のうち、適切なものを選ぶ形で問われます。

判断の中心は、導入対象が契約に含まれるか、契約外の追加依頼を勝手に引き受けていないか、作業中止や復旧について発注者と合意しているかです。

この判断軸により、契約外のPCを対象外とする行動、発注者と合意して導入前の状態に復旧する行動は適切です。一方、契約外のPCへの導入を無償で行う行動は不適切です。

まとめ

  • 導入作業では、契約に含まれる作業範囲を確認する。
  • 契約対象外のPCは、勝手に導入対象へ加えない。
  • 追加作業を行う場合は、作業範囲・費用・責任について合意が必要である。
  • 作業続行が困難な場合は、発注者と合意して中止や復旧を判断する。
