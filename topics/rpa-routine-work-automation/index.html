問22RPA

人の定型作業をソフトウェアに任せる仕組み

画面操作や転記などの定型的な事務作業を、ソフトウェアロボットで自動化する考え方を整理します。

トップ / 学習要点 / 人の定型作業をソフトウェアに任せる仕組み
学習要点一覧へ戻る

1場面をつかむ

会社の事務作業には、伝票入力、データ転記、定型フォームへの入力、帳票作成など、毎回ほぼ同じ手順で行う作業があります。

こうした定型作業を人が繰り返し行う代わりに、ソフトウェアに操作を覚えさせ、自動で実行させる仕組みが使われます。

2結論を先に押さえる

RPAとは、Robotic Process Automation の略で、定型的な業務をソフトウェアロボットによって自動化することです。

試験では、「人がPC上で行っている定型的な事務作業を、ソフトウェアロボットで自動化する」と押さえます。

3RPAの構造を確認する

RPAは、工場の機械ロボットではなく、PC上の操作を自動実行するソフトウェアの仕組みです。

  • 対象業務：手順が決まっている定型作業。
  • 操作内容：入力、転記、照合、集計、帳票作成など。
  • 実行主体：人ではなく、ソフトウェアロボット。
  • 目的：作業時間の短縮、ミスの削減、業務効率化。

4判断ポイントを整理する

RPAかどうかは、次の三つの観点で判断します。

  • 対象が定型的な事務作業である。
  • PC上の操作をソフトウェアが代行している。
  • 業務の効率化や自動化を目的としている。

5間違えやすい用語を分ける

  • BPO：業務プロセスを外部の事業者へ委託すること。
  • EUC：利用部門が自らシステムやツールを作成・利用すること。
  • FA：工場の生産工程を自動化すること。
  • RPA：定型的な事務作業をソフトウェアロボットで自動化すること。

6まとめで定着させる

  • RPAは、Robotic Process Automation の略である。
  • 定型的な事務作業をソフトウェアロボットで自動化する。
  • 入力、転記、照合などの繰り返し作業に向いている。
  • 試験では、外部委託や工場自動化と区別する。

補足：人の判断が必要な仕事とは分ける

RPAは、決まった手順を正確に繰り返す作業に向いています。

一方で、例外対応、判断、交渉、企画など、人の判断が大きく関わる業務は、そのままRPAで置き換えるのではなく、業務の切り分けが必要です。
