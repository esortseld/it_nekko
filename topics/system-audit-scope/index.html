マネジメント系 / システム監査

情報セキュリティ監査が検証する対象を見分ける

監査が、管理策や運用状況を客観的に確認する活動であることを整理します。

学習要点一覧へ戻る

この要点で学ぶこと

システム監査は、情報システムに関するリスクや統制を独立した立場で評価する活動です。情報セキュリティ監査では、機密性、完全性、可用性を守るための管理策が適切に整備され、運用されているかを確認します。

現実の場面と結び付けて考える概念

  • 監査は、客観的な証拠に基づいて評価する。
  • 情報セキュリティ監査では、セキュリティ管理策の整備状況と運用状況を見る。
  • 監査の結論は、確認した事実と基準に基づいて導く。

混同しやすいもの

  • システム監査：情報システム全般のリスクや統制を評価する。
  • 情報セキュリティ監査：セキュリティ管理策に焦点を当てる。
  • 運用作業：監査対象を実際に動かす作業であり、監査そのものではない。
  • 改善命令：監査人は通常、評価や助言を行い、業務執行そのものは担わない。

何を見ればこの概念だと分かるか

「監査では何を確認するか」と問われたら、基準、証拠、管理策、運用状況という語に注目します。監査人が自ら対策を実施する選択肢は慎重に見ます。
