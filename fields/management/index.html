マネジメント系

マネジメント系では、ITを作る・使う・任せる・確認する仕組みを学びます。用語だけで覚えるのではなく、会社で新しいシステムを入れる場面や、運用を管理する場面から整理します。

学習分野一覧へ戻る

このページでの分け方

マネジメント系は、公式には開発技術、プロジェクトマネジメント、サービスマネジメントなどに分かれます。ただ、問題を読むときに大事なのは、分類名を覚えることよりも、今どの場面の話をしているのかをつかむことです。

このページでは、公式分類を土台にしながら、システムを作る流れ、開発の進め方、プロジェクト管理、運用、監査など、問題を読むときの判断軸で9つの領域に分けています。

Topics

マネジメント系の学習要点

1. システムを作る流れ

要件定義、導入、使用性など、システムを作る前後の流れを整理します。

システム開発 / 要件定義

システム要件定義では利用者が妥当性を確認する

開発者が整理した要件を、利用者が業務の観点でレビューし妥当性を確認する役割を整理します。

関連する公開問題：令和8年度公開問題 問52

 システム開発 / 導入

完成したソフトウェアを本番環境に入れる工程を見分ける

開発済みのソフトウェアを利用できる状態にする導入・展開の考え方を整理します。

関連する公開問題：令和7年度公開問題 問46

 システム開発 / 使用性

使用性では利用者が操作するインタフェースを見る

利用者が分かりやすく、迷わず、効率よく操作できるかという使用性の観点を整理します。

関連する公開問題：令和7年度公開問題 問47

2. 開発の進め方

ウォーターフォール、アジャイル、プロトタイピングなど、開発の進め方を整理します。

システム開発 / 開発モデル

ウォーターフォールモデルの進め方を見分ける

工程を順番に進め、前工程の成果を次工程へ引き継ぐ開発モデルを整理します。

関連する公開問題：令和8年度公開問題 問38

 システム開発 / アジャイル

最初から全部決めず、作りながら確かめる

短い反復とフィードバックで、変化する要求に対応する開発の進め方を整理します。

関連する公開問題：令和7年度公開問題 問39 / 令和8年度公開問題 問43

 システム開発 / 開発モデル

試作品で利用者の要求を確認するプロトタイピングを見分ける

試作品を使って利用者の要求を確認し、認識違いを減らす開発の進め方を整理します。

関連する公開問題：令和7年度公開問題 問44

3. プロジェクトを始める

プロジェクト開始時の認可、契約範囲、前提条件を整理します。

プロジェクトマネジメント / 立上げ

プロジェクトを正式に認可する文書を見分ける

プロジェクト憲章を、プロジェクトの開始を正式に認可する文書として整理します。

関連する公開問題：令和8年度公開問題 問45

 プロジェクトマネジメント / 契約管理

契約に含まれないPCへ導入を求められたらどうするか

ソフトウェア導入作業を例に、契約範囲外の追加依頼、無償対応、発注者との合意、復旧判断を整理します。

関連する公開問題：令和8年度公開問題 問36

4. 作業・日程を管理する

作業分解、日程作成、作業順序、要員制約を整理します。

プロジェクトマネジメント / WBS

大きな仕事を、管理できる作業に分ける

作業の全体像を見えるようにし、見積りや担当割当ての土台にする考え方を整理します。

関連する公開問題：令和8年度公開問題 問39

 プロジェクトマネジメント / スケジュール

スケジュール作成に用いる技法を見分ける

作業順序、所要期間、依存関係を整理して日程を組み立てる技法を学びます。

関連する公開問題：令和7年度公開問題 問41

 プロジェクトマネジメント / スケジュール

前提作業と要員制約から最短日数を求める

作業の前後関係に加えて、同時に使える要員数の上限を考えるスケジュール計算を整理します。

関連する公開問題：令和8年度公開問題 問50

5. 進捗・コスト・人を管理する

見積り、進捗、コスト、人員や役割の管理を整理します。

プロジェクトマネジメント / 見積り

プロジェクト初期の概算見積りを考える

詳細情報が少ない段階で、過去の類似事例を基に概算する考え方を整理します。

関連する公開問題：令和8年度公開問題 問37

 プロジェクトマネジメント / 進捗管理

計画値と実績値からプロジェクトの進捗とコストを評価する

計画値、出来高、実績値を比較して、進み具合と費用の状態を判断する考え方を整理します。

関連する公開問題：令和7年度公開問題 問45

 プロジェクトマネジメント / 人的資源

プロジェクトの人的資源計画でスキルと役割を整理する

必要な要員、スキル、役割、責任、参画時期を整理する計画を学びます。

関連する公開問題：令和7年度公開問題 問55

 プロジェクトマネジメント / スコープ

プロジェクトスコープの対象になる成果物を見分ける

プロジェクトで作成する成果物と、それを完成させる作業範囲を整理します。

関連する公開問題：令和7年度公開問題 問48

 プロジェクトマネジメント / 会議運営

プロジェクト会議では決定事項を記録して共有する

決定事項、課題、担当、期限を議事録に残し、関係者へ共有する考え方を整理します。

関連する公開問題：令和8年度公開問題 問49

6. 利用者対応をする

利用者からの問合せ、受付、自動応答、対応時間の改善を整理します。

サービスマネジメント / サービスデスク

利用者の「困った」を最初に受け止める窓口

利用者対応の単一窓口として、問合せを記録し、一次対応や引継ぎを行う役割を整理します。

関連する公開問題：令和8年度公開問題 問42

 サービスマネジメント / チャットボット

チャットボットで問合せ対応の受付時間を広げる

会話形式で自動応答する仕組みを、サービスデスクや問合せ対応の文脈で整理します。

関連する公開問題：令和7年度公開問題 問49・問50

 サービスマネジメント / 自動化効果

AI導入による対応時間短縮率を計算する

導入前後の作業時間を同じ単位で比較し、自動化による削減率を求める考え方を整理します。

関連する公開問題：令和7年度公開問題 問54 / 令和8年度公開問題 問44

7. 障害・変更・品質を管理する

障害対応、原因分析、変更要求、SLA、可用性、継続的改善を整理します。

サービスマネジメント / 問題管理

まず使える状態に戻すのか、原因まで直すのか

迅速な復旧と、根本原因を調査して解決策を示す活動の違いを整理します。

関連する公開問題：令和8年度公開問題 問46

 プロジェクトマネジメント / 変更管理

変更要求は、影響を確認してから承認する

プロジェクトへの変更要求を、影響評価と承認の流れで管理する考え方を整理します。

関連する公開問題：令和7年度公開問題 問40

 サービスマネジメント / SLA

サービスの約束を、数値で決めて守る

サービス品質の合意であるSLAと、それを維持改善するSLMを分けて整理します。

関連する公開問題：令和7年度公開問題 問37 / 令和8年度公開問題 問53

 サービスマネジメント / 可用性

サービスを利用できた時間の割合を可用性として見分ける

合意したサービス提供時間のうち、実際に利用できた時間の割合として可用性を整理します。

関連する公開問題：令和7年度公開問題 問36

 サービスマネジメント / 継続的改善

サービスマネジメントシステムの改善に当たる対応を見分ける

応急処置や設備増強ではなく、状態を把握し早期検知できる管理の仕組みを整える考え方を整理します。

関連する公開問題：令和8年度公開問題 問48

8. 設備・環境を管理する

オフィス、設備、配線、施設資源など、サービスを支える環境を整理します。

サービスマネジメント / ファシリティ

ファシリティマネジメントで改善する対象を見分ける

オフィス、設備、レイアウトなどの施設資源を最適に活用する改善を整理します。

関連する公開問題：令和8年度公開問題 問54

 ファシリティ / 設備

床下配線用の空間をもつ床をフリーアクセスフロアとして見分ける

配線や設備変更に対応しやすい床構造として、フリーアクセスフロアを整理します。

関連する公開問題：令和7年度公開問題 問42

9. 第三者が確認する

システム監査、監査人の独立性、内部統制、ITガバナンスを整理します。

システム監査 / 監査プロセス

情報セキュリティ監査が検証する対象を見分ける

システム監査や情報セキュリティ監査で、管理策や運用状況を客観的に確認する考え方を整理します。

関連する公開問題：令和7年度公開問題 問38 / 令和8年度公開問題 問47・問51

 システム監査 / 独立性

システム監査人は監査対象から独立している必要がある

監査対象との担当関係や利害関係を避け、客観的に評価できる立場を整理します。

関連する公開問題：令和7年度公開問題 問51・問52

 システム監査 / 内部統制

内部統制におけるけん制とITへの対応を見分ける

業務を適切に進めるための仕組みとして、別部門による確認やITを利用した統制を整理します。

関連する公開問題：令和7年度公開問題 問43・問53 / 令和8年度公開問題 問40

 システム監査 / ITガバナンス

新しいシステムを入れるとき、経営陣は何をするのか

勤怠システムの入れ替えを例に、ITガバナンスにおける評価・指示・モニターを整理します。

関連する公開問題：令和8年度公開問題 問35・問41
