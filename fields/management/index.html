マネジメント系

マネジメント系では、ITを作る・使う・任せる・確認する仕組みを学びます。用語だけで覚えるのではなく、会社で新しいシステムを入れる場面や、運用を管理する場面から整理します。

Topics

マネジメント系の学習要点

システム監査 / ITガバナンス

新しいシステムを入れるとき、経営陣は何をするのか

勤怠システムの入れ替えを例に、ITガバナンスにおける評価・指示・モニターを整理します。

この要点で解ける問題：令和8年度公開問題 問35・問41

 プロジェクトマネジメント / 契約管理

契約に含まれないPCへ導入を求められたらどうするか

ソフトウェア導入作業を例に、契約範囲外の追加依頼、無償対応、発注者との合意、復旧判断を整理します。

この要点で解ける問題：令和8年度公開問題 問36

 プロジェクトマネジメント / 見積り

プロジェクト初期の概算見積りを考える

詳細情報が少ない段階で、過去の類似事例を基に概算する考え方を整理します。

この要点で解ける問題：令和8年度公開問題 問37

 システム開発 / 開発モデル

ウォーターフォールモデルの進め方を見分ける

工程を順番に進め、前工程の成果を次工程へ引き継ぐ開発モデルを整理します。

この要点で解ける問題：令和8年度公開問題 問38

 プロジェクトマネジメント / WBS

WBSは成果物を作る作業を階層的に分解したもの

作業の全体像を見えるようにし、見積りや担当割当ての土台にする考え方を整理します。

この要点で解ける問題：令和8年度公開問題 問39

 システム監査 / 内部統制

内部統制は業務を適切に進めるための仕組み

業務の有効性、法令遵守、資産保全、情報の信頼性を支える統制を整理します。

この要点で解ける問題：令和8年度公開問題 問40

 サービスマネジメント / サービスデスク

サービスデスクは利用者からの問合せや障害連絡を受け付ける窓口

利用者対応の単一窓口として、問合せを記録し、一次対応や引継ぎを行う役割を整理します。

この要点で解ける問題：令和8年度公開問題 問42

 システム開発 / アジャイル

アジャイル開発は短い反復で改善しながら進める

変化に対応しやすい開発の進め方として、反復とフィードバックの考え方を整理します。

この要点で解ける問題：令和8年度公開問題 問43

 サービスマネジメント / 自動化効果

サービスデスクの自動化による作業時間削減を計算する

導入前後の作業時間を同じ単位で比較し、自動化による削減率を求める考え方を整理します。

この要点で解ける問題：令和8年度公開問題 問44

 プロジェクトマネジメント / 立上げ

プロジェクトを正式に認可する文書を見分ける

プロジェクト憲章を、プロジェクトの開始を正式に認可する文書として整理します。

この要点で解ける問題：令和8年度公開問題 問45

 サービスマネジメント / 問題管理

インシデント管理と問題管理の違いを見分ける

迅速な復旧と、根本原因を調査して解決策を示す活動の違いを整理します。

この要点で解ける問題：令和8年度公開問題 問46

 システム監査 / 監査プロセス

システム監査が検証する範囲と流れを見分ける

システム監査の対象、情報セキュリティ監査との違い、評価・結論の段階を整理します。

この要点で解ける問題：令和8年度公開問題 問47・問51

 サービスマネジメント / 継続的改善

サービスマネジメントシステムの改善に当たる対応を見分ける

応急処置や設備増強ではなく、状態を把握し早期検知できる管理の仕組みを整える考え方を整理します。

この要点で解ける問題：令和8年度公開問題 問48

 プロジェクトマネジメント / 会議運営

プロジェクト会議では決定事項を記録して共有する

決定事項、課題、担当、期限を議事録に残し、関係者へ共有する考え方を整理します。

この要点で解ける問題：令和8年度公開問題 問49

 プロジェクトマネジメント / スケジュール

前提作業と要員制約から最短日数を求める

作業の前後関係に加えて、同時に使える要員数の上限を考えるスケジュール計算を整理します。

この要点で解ける問題：令和8年度公開問題 問50

 システム開発 / 要件定義

システム要件定義では利用者が妥当性を確認する

開発者が整理した要件を、利用者が業務の観点でレビューし妥当性を確認する役割を整理します。

この要点で解ける問題：令和8年度公開問題 問52

 サービスマネジメント / SLA

SLAを計画し合意し測定する活動を見分ける

SLAを作成・合意・測定し、サービス品質を維持改善するサービスレベル管理を整理します。

この要点で解ける問題：令和8年度公開問題 問53

 サービスマネジメント / ファシリティ

ファシリティマネジメントで改善する対象を見分ける

オフィス、設備、レイアウトなどの施設資源を最適に活用する改善を整理します。

この要点で解ける問題：令和8年度公開問題 問54
